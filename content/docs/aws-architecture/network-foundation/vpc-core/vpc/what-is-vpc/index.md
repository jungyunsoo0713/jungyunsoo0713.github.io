# VPC란?

Virtual Private Cloud(이하 VPC)는 AWS를 학습할 때 일반적으로 가장 먼저 배우는 핵심 인프라입니다. 직역하면 가상의 개인 클라우드라는 뜻입니다. VPC는 방대한 AWS 네트워크의 일부를 할당받아, 자신만의 격리된 공간에서 애플리케이션을 안전하게 운영할 수 있도록 해주는 서비스입니다.

## VPC는 왜 필요한가요?

애플리케이션은 보통 사설 네트워크에서 운영됩니다. 그래야 외부의 해킹 위협으로부터 보다 안전할 수 있고, 운영자도 애플리케이션에 대한 통제권을 확보할 수 있기 때문입니다. VPC 역시 이러한 사설 네트워크 역할을 하므로, 클라우드에서 애플리케이션을 운영하려면 VPC가 필요합니다.

## VPC는 어떻게 생성하나요?

앞서 말했듯이, VPC는 사설 네트워크입니다. 따라서 사설 네트워크에 해당하는 IP 주소 범위를 가져야 합니다. VPC가 하나의 주소값이 아니라 주소 범위를 가져야 하는 이유는, VPC 내부에 배치되는 인프라들도 VPC에 속한 주소 범위 안에서 각자의 Private IP를 할당받기 때문입니다.

사설 네트워크에서 사용할 수 있는 IP 주소 범위는 이미 정해져 있습니다. 따라서 VPC를 생성할 때는 아래와 같은 사설 IP 주소 범위 중 하나를 선택해야 합니다.

사설(Private) IP 주소 범위는 RFC 1918에서 정의한 다음 세 가지입니다.

|클래스|대역|CIDR|주소 수|
|---|---|---|---|
|A|10.0.0.0 ~ 10.255.255.255|10.0.0.0/8|약 1677만 개|
|B|172.16.0.0 ~ 172.31.255.255|172.16.0.0/12|약 104만 개|
|C|192.168.0.0 ~ 192.168.255.255|192.168.0.0/16|약 6만 5천 개|

AWS에서 VPC를 생성할 때는 일반적으로 `10.0.0.0/16`이나 `172.16.0.0/16`과 같은 사설 IP 주소 범위를 많이 사용합니다

### CIDR 표기법이란 무엇인가요?

앞서 언급한 주소 범위 예시인 `10.0.0.0/16`이나 `172.16.0.0/16`은 CIDR이라는 IP 주소 표기법을 사용한 것입니다. VPC가 사용할 주소 범위의 크기를 파악하려면 이 CIDR 표기법을 먼저 이해해야 합니다.

IP 주소는 네 개의 숫자를 세 개의 점으로 구분해 표현하며, 각 숫자는 실제로 8비트 이진수에 해당합니다. 예를 들어 `10.0.0.0`은 이진수로 `00001010.00000000.00000000.00000000`입니다.

CIDR 표기인 `10.0.0.0/16`에서 `/16`은 앞의 16비트를 네트워크를 식별하는 데 사용한다는 뜻입니다. 따라서 뒤의 16비트가 변할 수 있으며, 이 네트워크가 표현하는 주소 범위는 `00001010.00000000.00000000.00000000`부터 `00001010.00000000.11111111.11111111`까지입니다. 즉, 이를 십진수로 나타내면 `10.0.0.0`부터 `10.0.255.255`까지입니다.

`/` 뒤의 숫자가 클수록 네트워크를 식별하는 데 사용하는 비트 수가 많아지므로, IP 주소 범위를 나타내는 데 사용할 수 있는 비트 수는 줄어듭니다. 즉, `/` 뒤의 숫자가 클수록 표현할 수 있는 IP 주소 범위는 더 좁아집니다. `/16`에서는 전체 32비트 중 앞의 16비트가 네트워크를 식별하는 데 사용되고, 나머지 16비트가 IP 주소 범위를 표현하는 데 사용됩니다. 따라서 `2^16`, 즉 65,536개의 IP 주소를 표현할 수 있습니다.

### VPC 생성 시 주의할 점이 있나요?

서로 다른 독립적인 VPC나 사설 네트워크를 연결해 통신할 때는 IP 주소 범위가 겹치면 안 됩니다. 서로 다른 인프라가 같은 IP 주소를 사용하면, 네트워크가 어느 인프라로 통신해야 하는지 판단할 수 없기 때문입니다.

또한 IP 주소 범위는 신중하게 정해야 합니다. 주소 범위가 너무 좁으면 내부 인프라에 할당할 IP 주소가 부족할 수 있고, 반대로 너무 넓으면 사용하지 않는 IP 주소 범위가 많아져 비효율적일 수 있습니다.
