# Route Table이란?

Route Table(라우트 테이블)은 서브넷에 위치한 리소스가 보내는(나가는) 트래픽을 목적지에 따라 어디로 전달할지 정하는 테이블입니다. 예를 들어, 어떤 리소스가 인터넷이나 VPC 내부의 다른 리소스로 트래픽을 보내야 할 때, 라우트 테이블은 그 트래픽의 경로를 정합니다. 
## Route Table은 왜 필요한가요?

서브넷에 위치한 리소스가 내보내는 트래픽은 목적지까지 전달되어야 하기 때문입니다. 트래픽에는 목적지의 IP 주소가 적혀 있습니다. 하지만 트래픽은 스스로 이동할 수 없으므로 AWS의 네트워크 시스템이 이를 전달해야 합니다. 이때 라우트 테이블은 목적지 IP 주소를 기준으로 해당 트래픽을 어디로 보낼지 결정하는 역할을 합니다.

## Route Table 예시

라우트 테이블에는 리소스가 내보내는 트래픽의 목적지에 따라 어느 경로로 보낼지 정한 규칙들이 들어 있습니다.

다음은 라우트 테이블의 예시입니다:

| 목적지(Destination) | 대상(Target)     | 의미                                 |
| ---------------- | -------------- | ---------------------------------- |
| `10.0.0.0/16`    | `local`        | 같은 VPC 내부 리소스로 가는 트래픽은 VPC 내부에서 전달 |
| `0.0.0.0/0`      | `igw-1234abcd` | 그 외 모든 외부 트래픽은 인터넷 게이트웨이로 전달       |

## Route Table의 작동 원리

리소스가 내보내는 트래픽의 목적지가 `10.0.0.0/16` IP 주소 범위에 포함된다면, 라우트 테이블은 그 트래픽에 대해 `local` 경로를 선택합니다. 그 후 AWS의 네트워크 시스템은 선택된 경로를 따라 트래픽을 전달합니다.

반면, 리소스가 내보내는 트래픽의 목적지가 `10.0.0.0/16` IP 주소 범위에 포함되지 않는다면, 라우트 테이블은 그 트래픽에 대해 인터넷 게이트웨이로 향하는 경로를 선택합니다. 그 후 해당 트래픽은 선택된 경로를 따라 전달됩니다. 여기서 `0.0.0.0/0`은 모든 목적지 IP 주소를 뜻합니다. 따라서 목적지 IP 주소가 `10.0.0.0/16` 범위에 속하지 않는 트래픽은 `0.0.0.0/0` 규칙에 매치됩니다.


## 퍼블릭 서브넷과 프라이빗 서브넷의 라우트 테이블은 어떻게 다른가요?

퍼블릭 서브넷의 라우트 테이블 예시:

|목적지(Destination)|대상(Target)|의미|
|---|---|---|
|`10.0.0.0/16`|`local`|같은 VPC 내부 리소스로 가는 트래픽은 VPC 내부에서 전달|
|`0.0.0.0/0`|`igw-1234abcd`|그 외 모든 외부 트래픽은 인터넷 게이트웨이로 전달|

프라이빗 서브넷의 라우트 테이블 예시:

|목적지(Destination)|대상(Target)|의미|
|---|---|---|
|`10.0.0.0/16`|`local`|같은 VPC 내부 리소스로 가는 트래픽은 VPC 내부에서 전달|
|`0.0.0.0/0`|`nat-1234abcd`|외부로 나가는 트래픽은 NAT 게이트웨이로 전달|

퍼블릭 서브넷에 위치한 리소스의 나가는 트래픽이 `local` 경로에 해당하지 않는다면, 이 트래픽은 인터넷 게이트웨이(IGW)로 전달됩니다. 반면, 프라이빗 서브넷에 위치한 리소스의 나가는 트래픽이 `local` 경로에 해당하지 않는 경우에는 NAT 게이트웨이(NAT GW)로 전달됩니다. 인터넷 게이트웨이와 NAT 게이트웨이는 모두 인터넷으로 향하는 트래픽 전달에 관여하지만, 그 역할과 존재 이유는 다릅니다. 이에 대해서는 추후 포스트에서 다룰 예정입니다.
