## Amazon ECS Core Components

ECS Cluster: 서비스들과 태스크들의 논리적 집합
Service: 동일한 태스크들의 집합
Task: 하나 이상의 컨테이너를 실행하는 단위
Task Definition: 태스크의 설계도

## ECS Service

ECS Service는 ECS Cluster에서 특정 태스크 수를 실행하고 유지하기 위해 사용됩니다. 이를 위해 ECS Service를 생성하기 전에 Task Definition을 먼저 생성해야 합니다.

### ECS Service의 역할

1. 동일한 태스크의 집합이며, 태스크를 프로비저닝합니다.
2. 비정상적인 태스크를 감지하고 교체합니다.
3. ECS의 Scheduling Strategy로 Replica와 Daemon 방식이 있습니다.
Scheduling Strategy는 태스크를 배치하고 유지하는 방식을 의미합니다.

Replica는 원하는 개수만큼 태스크를 유지하는 방식입니다. 태스크의 개수를 지정하면 서비스가 이 태스크 개수를 유지하려고 합니다. 일반적인 웹 애플리케이션은 이 방식을 주로 적용하며, 이 값은 기본값입니다.

Daemon은 각 EC2 컨테이너 인스턴스마다 해당 서비스의 태스크를 1개씩 띄우는 방식입니다. Daemon 방식은 동일한 애플리케이션을 여러 개 띄우는 용도보다는 각 EC2 인스턴스마다 꼭 하나씩 떠 있어야 하는 보조 태스크(로그 수집기, 모니터링 에이전트, 보안 에이전트 등)에 주로 적용합니다. Fargate는 컨테이너 인스턴스가 없기 때문에 이 방식을 지원하지 않습니다. EC2 런치 타입만 지원합니다. 

>하나의 인스턴스에 여러개의 서비스의 태스크들이 실행될 수 있기 때문에, 하나의 인스턴스에 어떤 서비스는 Replica 방식으로 태스크 수를 유지할 수 있고, 다른 서비스는 Daemon 방식으로 하나의 태스크를 배치할 수 있습니다. 다음은 예시 구조입니다.

```
EC2 Instance A
├─ ui 서비스의 Task          ← Replica 방식
├─ orders 서비스의 Task      ← Replica 방식
├─ checkout 서비스의 Task    ← Replica 방식
└─ log-agent 서비스의 Task   ← Daemon 방식
```

>참고로 엄밀히 말하자면, 인스턴스에서 실행되는 것은 서비스가 아니라 서비스의 태스크입니다.

>태스크의 `awsvpc` 네트워크 모드에서는 각 태스크의 ENI에 할당된 IP가 모두 다릅니다. ALB를 통해 들어온 요청은 태스크의 IP와 컨테이너 포트로 전달되고, 해당 컨테이너가 요청을 받습니다. 이때 컨테이너 포트가 동일하더라도 태스크의 IP가 다르기 때문에, 같은 EC2 인스턴스 안에서도 여러 동일한 태스크가 같은 컨테이너 포트 번호를 사용할 수 있습니다.

```
10.0.1.10 / 10.0.2.10 / 10.0.3.10 = 각 태스크 ENI에 할당된 IP
8080 = 태스크 안 컨테이너의 컨테이너 포트

10.0.1.10:8080 
10.0.2.10:8080
10.0.3.10:8080
```
>이 예시는 서로 다른 서브넷에 동일한 태스크를 배치한 예시입니다. 같은 서브넷에 여러 태스크를 배치하는 경우도 가능합니다.

>ECS에서는 EC2 인스턴스 주소는 일반적인 애플리케이션 트래픽 처리에 잘 사용되지 않습니다. 사용자로부터 트래픽을 받고 응답하는 기본적인 애플리케이션 운영에는 ALB와 태스크의 IP 주소가 사용됩니다.

4. 오토 스케일링과 배포 방식에 관여합니다.

## ECS 태스크와 Task Definition

ECS 태스크는 ECS 클러스터에서 실행 가능한 가장 작은 단위로, 하나 이상의 컨테이너를 가질 수 있습니다. 태스크는 Task Definition을 기반으로 생성되며, Task Definition을 생성하는 것은 ECS 클러스터에서 태스크를 실행하기 위해 가장 먼저 해야 할 일입니다.

Task Definition에서는 다음과 같은 파라미터를 설정할 수 있습니다.

1. 하나 이상의 컨테이너 이미지
2. CPU/Memory 사양
3. Launch Type(태스크를 어떤 환경에서 실행할지 정하는 방식) - EC2, Fargate, 혹은 둘 다
4. 네트워크 모드 - `awsvpc`, `bridge`, `host`, `none`, `default` 등(단, Fargate 실행 타입에서는 `awsvpc` 설정만 사용할 수 있습니다.)
